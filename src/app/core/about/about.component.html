On Security ğŸ”

We use TeamCity's token-based authentication. This is a variety of basic authentication. Therefore we need to store your token on the client and send it along with every request. For your security we suggest the following:

  • Set up a read only user in TeamCity for this build screen.
  • Generate a token for that user and use it to sign in. This token will be stored in your browser's localstorage and is readable and therefore steal-able by javascript. But since it is a read only user your exposure is limited ğŸ˜‡
  • However, if your build data contains or is considered sensitive information, don't have a build screen ğŸ™€
